# 网页端学院公告与提醒邮件发送功能实现说明

## 1. 功能目标

在网页端“学院公告与提醒”模块中，为管理员发送通知新增发送方式选择：

1. 网站内发送：保留原有站内通知能力。
2. 邮件发送：通过系统邮箱向学生邮箱发送邮件通知。

邮件收件地址按以下规则生成：

```text
学号@ruc.edu.cn
```

## 2. 适用范围

本次仅覆盖网页端管理员角色：

- 页面：`webpage/src/pages/Reminder.jsx`
- 后端：`server/index.js`
- 邮件模块：`server/mail.js`

不覆盖：

1. 小程序端邮件发送。
2. 学生端邮件订阅偏好。
3. 邮件模板后台可配置。
4. 邮件附件发送。
5. 失败邮件自动重试。

## 3. 当前实现边界

### 3.1 网站内发送

网站内发送沿用原逻辑：

1. 发送给全部学生。
2. 按标签发送。

发送后会写入站内通知表，学生可在网页端提醒列表中查看。

### 3.2 邮件发送

邮件发送支持三种范围：

1. 发送给全部学生。
2. 按标签发送。
3. 批量选择。

批量选择规则：

1. 页面滚动框中展示所有启用的学生账号。
2. 管理员手动勾选目标学生。
3. 后端仍会校验所选账号是否为启用学生账号。
4. 邮件发送至 `学号@ruc.edu.cn`。

## 4. 不清晰点与当前取舍

### 4.1 邮件是否同步生成站内通知

当前取舍：

- 邮件发送不写入站内通知列表。
- 网站内发送和邮件发送是两个独立通道。

原因：

1. 用户明确要求新增“通知发送方式选择”，因此邮件发送应作为单独方式处理。
2. 若邮件发送同时写入站内通知，可能造成用户认为收到了两条通知。

后续如需要“双通道同时发送”，建议新增第三个发送方式，而不是隐式叠加。

### 4.2 5 分钟限制的作用范围

当前取舍：

- 所有管理员共享一个邮件发送冷却时间。
- 任意一次邮件发送请求后，5 分钟内不允许再次发送邮件通知。

原因：

1. 该功能可能触发大批量邮件，风险主要在系统邮箱发信频率。
2. 共享限流比按管理员限流更能保护 SMTP 账号。

### 4.3 部分邮件发送失败如何处理

当前取舍：

1. 后端逐个发送邮件。
2. 记录成功数和失败数。
3. 只要至少一封发送成功，接口返回成功并附带失败数。
4. 如果全部失败，接口返回失败。

当前不做自动重试，避免在 SMTP 异常时重复刷信。

### 4.4 邮件历史是否展示

当前实现：

- 后端记录邮件发送日志。
- 前端暂不展示邮件发送历史。

后续可在管理员端补充“邮件发送记录”列表。

## 5. 后端实现

### 5.1 新增邮件发送函数

文件：

- `server/mail.js`

新增函数：

```js
sendReminderEmail({ to, title, content })
```

邮件标题格式：

```text
第3组学生服务平台通知：{通知标题}
```

邮件正文包含：

1. 问候语。
2. 平台来源说明。
3. 通知标题。
4. 通知内容。
5. 联系管理员提示。

### 5.2 新增邮件发送日志表

表名：

```text
reminder_email_logs
```

用途：

1. 记录邮件发送审计信息。
2. 记录目标范围。
3. 记录成功数和失败数。
4. 作为 5 分钟限流依据。

主要字段：

1. `title`
2. `content`
3. `target_type`
4. `target_tags`
5. `target_accounts`
6. `recipient_count`
7. `success_count`
8. `fail_count`
9. `created_by`
10. `created_at`
11. `finished_at`
12. `status`
13. `error`

### 5.3 新增接口

接口：

```http
POST /api/reminder/admin/email
```

权限：

- 仅管理员可调用。

请求示例：

```json
{
  "title": "通知标题",
  "content": "通知内容",
  "targetType": "batch",
  "targetTags": [],
  "targetAccounts": ["2024201575"]
}
```

`targetType` 可选值：

1. `all`
2. `tags`
3. `batch`

后端校验：

1. 标题不能为空。
2. 内容不能为空。
3. 标题不超过 120 字。
4. 内容不超过 5000 字。
5. 按标签发送时必须选择标签。
6. 批量发送时必须选择学生。
7. 收件人必须是启用的学生账号。
8. 单次发送人数不能超过 `REMINDER_EMAIL_BATCH_LIMIT`。
9. 5 分钟内只能发送一次邮件通知。

## 6. 前端实现

页面：

- `webpage/src/pages/Reminder.jsx`

新增交互：

1. 通知发送方式选择。
2. 网站内发送：保留“全部学生”和“按标签发送”。
3. 邮件发送：支持“全部学生”“按标签发送”“批量选择”。
4. 批量选择时展示学生滚动列表。
5. 发送邮件前弹窗确认。
6. 邮件发送后显示成功数和失败数。

批量选择展示字段：

1. 姓名。
2. 学号。
3. 标签。
4. 目标邮箱。

## 7. 安全边界

### 7.1 权限边界

邮件发送接口必须经过：

1. 登录校验。
2. 管理员权限校验。

前端显示控制不是安全边界，后端校验才是最终边界。

### 7.2 收件人边界

后端只允许发送给：

```sql
permitted_accounts.role = 'student'
AND permitted_accounts.enabled = TRUE
```

即使前端提交管理员账号或伪造账号，后端也会过滤。

### 7.3 频率边界

邮件通知 5 分钟内仅允许发送一次。该限制在后端强制执行。

默认配置：

```env
REMINDER_EMAIL_INTERVAL_MS=300000
REMINDER_EMAIL_BATCH_LIMIT=2000
```

### 7.4 内容边界

当前实现限制：

1. 邮件标题最多 120 字。
2. 邮件内容最多 5000 字。
3. 不支持附件。
4. 不支持 HTML 邮件，只发送纯文本邮件。

### 7.5 日志边界

邮件发送日志不应记录 SMTP 授权码。

当前日志记录：

1. 通知标题。
2. 通知内容。
3. 目标类型。
4. 目标账号。
5. 成功数与失败数。
6. 简要错误信息。

## 8. 部署注意事项

服务器后端 `.env` 应放在：

```bash
~/software4infotuanwei/server/.env
```

需要确保已有 SMTP 配置：

```env
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=你的QQ邮箱
SMTP_PASS=你的QQ邮箱SMTP授权码
SMTP_FROM="第3组学生服务平台 <你的QQ邮箱>"
```

新增可选配置：

```env
REMINDER_EMAIL_INTERVAL_MS=300000
REMINDER_EMAIL_BATCH_LIMIT=2000
```

后端重启后会自动创建 `reminder_email_logs` 表。

## 9. 验收用例

1. 管理员选择“网站内发送”，发送给全部学生，学生端可看到站内通知。
2. 管理员选择“网站内发送”，按标签发送，只有匹配标签学生可看到站内通知。
3. 管理员选择“邮件发送”，发送给全部学生，所有启用学生邮箱收到邮件。
4. 管理员选择“邮件发送”，按标签发送，匹配标签学生邮箱收到邮件。
5. 管理员选择“邮件发送”，批量选择单个学生，仅该学生邮箱收到邮件。
6. 邮件发送前出现确认弹窗。
7. 取消确认后不发送邮件。
8. 邮件发送后 5 分钟内再次发送，应提示发送过于频繁。
9. 批量选择为空时不能发送。
10. 普通学生无法调用邮件发送接口。
11. 伪造非学生账号作为批量收件人时，后端不会向其发送邮件。

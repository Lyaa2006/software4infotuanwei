# 网页端荣誉页上传放开后的显示修正记录

日期：2026-06-08

## 背景

网页端荣誉页放开图片上传后，为了让“本人主页管理”能看到未公开荣誉，数据源从公开详情接口切换为：

- 条目列表：`GET /api/honor/me`
- 上传图片：`POST /api/honor/me/upload`

这次切换带来两个显示层遗漏：

1. `GET /api/honor/me` 只返回荣誉条目，不返回 `user.name`，导致页面标题可能退化为学号。
2. 本人视图已展示“我的全部荣誉”，但空态文案仍写成“暂无公开荣誉”，语义错误。

## 处理逻辑

网页端荣誉页现在按视图类型拆分处理：

1. 本人可编辑视图  
   - 用 `GET /api/honor/me` 拉取全部个人荣誉，确保未公开荣誉也能编辑。  
   - 再用 `GET /api/auth/me` 补充当前登录用户的 `name / accountId`，只用于页面标题展示。  
   - 空态文案显示为“你还没有添加荣誉记录。”

2. 他人公开视图  
   - 继续用 `GET /api/honor/users/:accountId` 拉取公开荣誉与用户展示信息。  
   - 空态文案保持为“暂无公开荣誉。”

## 结果

这样处理后：

- 本人查看荣誉主页时，能同时满足“看到全部可编辑条目”和“标题优先显示姓名”。
- 访客查看公开主页时，仍保持公开展示语义不变。
- 页面文案与实际数据权限范围保持一致，避免后续再次被旧接口假设影响。

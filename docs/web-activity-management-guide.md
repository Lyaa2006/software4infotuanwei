# Web 端班团活动管理说明

## 页面入口

网页端班团活动管理入口为 `/activity`。管理员可在该页面审核活动提案，并维护学生的“班团骨干”标签；学生账号可查看自己参与的活动，班团骨干可创建和维护自己的活动提案。

## 创建和编辑活动

只有学生账号可以进入学生侧活动功能。创建活动提案需要该学生拥有“班团骨干”标签。活动提案中的组织者、参与者和协助者必须都是真实学生账号；如果手动填写管理员账号或其他非学生账号，后端会拒绝保存。

待审核或已驳回的活动提案可以由创建它的班团骨干编辑。已审核通过的活动不可再编辑。

## 删除活动提案

“班团骨干申请 / 我的提案”列表中，待审核或已驳回等可编辑提案会显示“删除”按钮。点击删除前会弹出确认提示，确认后系统会调用后端删除接口并刷新列表；删除成功后该提案会从列表中消失，刷新页面后也不会恢复。

已审核通过的活动不可删除。无权限用户、非创建者或非班团骨干尝试删除时，后端会拒绝请求，页面会显示失败提示。

## 骨干设置

管理员仍可在 `/activity` 的学生列表中执行“设为骨干 / 移除骨干”。该列表只展示学生账号，管理员账号不会作为可设置骨干的对象出现。骨干身份仍通过学生标签中的“班团骨干”维护。

## 异常处理

如果删除失败，请先确认当前账号是否仍是该提案的创建者、是否仍拥有“班团骨干”标签，以及活动是否已经审核通过。若页面显示 `NOT_STUDENT_ACCOUNT`，说明目标账号不是学生账号，不能参与学生侧活动数据操作。

# 更新指南

## 前端更新

适用于：只改了 `webpage/` 里的页面、样式、前端逻辑。


```bash
tmux attach -t webpage #进入tmux，ctrlc终止之前的指令
cd ~/software4infotuanwei
git pull

cd ~/software4infotuanwei/webpage
npm install
npm run build

sudo rsync -av --delete dist/ /var/www/software4infotuanwei/
sudo nginx -t
sudo systemctl reload nginx
```

验证：

```bash
curl -I http://127.0.0.1
```

额外检查：

```bash
ls -l /etc/nginx/sites-enabled/software4infotuanwei
```

说明：

- `sites-enabled/software4infotuanwei` 应当是指向 `sites-available/software4infotuanwei` 的软链接
- 如果它是一个独立旧文件，改了 `sites-available` 也不会生效

如果发现不是软链接，执行：

```bash
sudo rm /etc/nginx/sites-enabled/software4infotuanwei
sudo ln -s /etc/nginx/sites-available/software4infotuanwei /etc/nginx/sites-enabled/software4infotuanwei
sudo nginx -t
sudo systemctl reload nginx
```

## 后端更新

适用于：只改了 `server/` 里的接口、数据库逻辑、模板逻辑。

如果你现在是手动在 `tmux` 里跑后端：

```bash
tmux 
cd ~/software4infotuanwei
git pull

cd ~/software4infotuanwei/server
npm install
```

然后进入后端那个 `tmux` 会话：

```bash
tmux attach -t server
```

在里面按：

```text
Ctrl + C
```

再重启：

```bash
npm start
```

然后退出但不停止：

```text
Ctrl+b d
```

验证：

```bash
curl http://127.0.0.1:3001/health
```

如果有问题，额外检查：

```bash
cat ~/software4infotuanwei/server/.env
```

重点确认这几项没有被误改回旧值：

```env
PORT=3001
TOKEN_SECRET=你的实际密钥
DB_HOST=127.0.0.1
DB_PORT=5432
DB_USER=system
DB_PASSWORD=123456
DB_NAME=student_service_platform
TOKEN_EXPIRES_MS=604800000
NODE_ENV=production
PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser
```

注意：

- `TOKEN_SECRET` 不要带空格，不要带单引号
- 当前服务器实际连库账号是 `system`，不是 `postgres`

```bash

```

然后重启后端：

```bash
tmux attach -t server
```

在会话里：

```text

```


```text
Ctrl+b d
```

最后验证：

```bash
curl http://127.0.0.1:3001/health

```

建议附带检查：

```bash
ls -l /etc/nginx/sites-enabled/software4infotuanwei

```

## Cloudflare Tunnel 相关

如果你现在用的是 Quick Tunnel：

- 不一定每次更新都要重开
- 但如果 tunnel 断了，就重新执行：

你是在 `tmux` 的 `cloudflare` 会话里跑：

```bash
tmux attach -t cloudflare
```

断了就重新执行那条命令，再：

```text
Ctrl+b d
```

注意：

- Quick Tunnel 不是固定地址
- 每次重新启动后，`trycloudflare.com` 地址通常会变化
- 如果 tunnel 重启了，需要把新地址同步给其他成员

## 建议记住的检查命令

看 `tmux`：

```bash
tmux ls
```

看后端端口：

```bash
ss -ltnp | grep 3001
```

看 Nginx：

```bash
sudo systemctl status nginx
```

看 Cloudflare 进程：

```bash
ps -ef | grep cloudflared
```

看 Nginx 报错：

```bash
sudo tail -n 50 /var/log/nginx/error.log
```

看站点配置是否统一：

```bash
sudo grep -R "root " /etc/nginx/sites-enabled /etc/nginx/sites-available
```

## 当前部署结构备注

当前网页静态文件由 Nginx 提供，实际目录是：

```text
/var/www/software4infotuanwei
```

所以前端每次更新后，必须重新：

1. `npm run build`
2. `rsync` 到 `/var/www/software4infotuanwei`

后端则不需要复制静态文件，只需要在 `server/` 里安装依赖并重启进程。

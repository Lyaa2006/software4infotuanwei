# Web 端学生标签管理指南

## 适用范围

学生标签管理页面用于维护真实学生账号的业务标签，例如“班团骨干”、通知分组标签、活动目标标签等。管理员账号、教师账号、禁用账号或权限清单中不存在的账号，都不属于学生标签管理对象。

本页面入口为 `/tag-management`，只有管理员可以进入。管理员的身份用于管理学生数据，但管理员账号本身不会作为学生出现在列表中。

## 学生账号范围

系统以 `permitted_accounts` 作为账号权限清单。真实学生账号需要满足：

- `role='student'`
- `enabled=TRUE`
- `account_id` 与页面/接口中的目标账号一致

`party_students` 只保存党团进度、姓名、标签等扩展资料，不是学生总表。即使历史数据中存在管理员账号的 `party_students` 行，也不会被当作学生展示或编辑。

## 页面入口

1. 使用管理员账号登录 Web 端。
2. 回到首页 Dashboard。
3. 在“班团与管理”分组中进入“学生标签管理”。
4. 浏览器路径为 `/tag-management`。

## 页面结构

页面包含三个主要区域：

1. 顶部工具栏：显示页面标题和返回首页按钮。
2. 学生列表卡片：展示可管理的学生账号、姓名、当前标签和编辑按钮。
3. 编辑弹窗：用于修改某个学生的标签。

页面会显示当前可用标签汇总。可用标签来自当前真实学生账号的标签集合，不包含管理员账号的历史脏标签。

## 标签编辑流程

1. 在学生列表中找到目标学生。
2. 点击“编辑标签”。
3. 在弹窗中确认学号、姓名和当前标签。
4. 输入新的标签文本。
5. 点击“保存”。
6. 保存成功后弹窗关闭，列表刷新。

保存失败时页面会显示后端返回的错误信息。若目标账号不是学生账号，后端会返回 `NOT_STUDENT_ACCOUNT`。

## 标签输入规则

多个标签可以用以下任意方式分隔：

- 英文逗号
- 中文逗号
- 顿号
- 空格
- 换行

重复标签会自动合并。清空输入后保存，表示该学生暂无标签。

## 和 Reminder 的关系

Reminder 页面只负责选择通知发送范围，不负责编辑学生标签。按标签发送时，Reminder 会读取学生标签管理中的标签集合。

如果 `/reminder` 没有可用标签，请先到 `/tag-management` 为真实学生维护标签。

## 和 Activity 的关系

Activity 页面保留“设为骨干 / 移除骨干”功能。该功能本质上维护学生标签中的“班团骨干”，用于判断学生是否可以创建活动提案。

Activity 的骨干设置列表同样只展示真实学生账号。管理员账号不会成为骨干设置对象。

## 常见问题

### 为什么管理员账号不显示在学生标签管理列表中？

管理员不是学生管理对象。管理员可以管理学生数据，但管理员账号自身不能被编辑学生标签。

### 如果历史数据里已经有管理员标签怎么办？

本次修复不会删除历史数据，但接口和页面都会过滤非学生账号。历史脏数据不会再显示为学生，也不能继续被学生标签接口编辑。

### 直接调用接口能否给管理员编辑标签？

不能。后端会校验目标账号是否是 `permitted_accounts` 中启用的学生账号。非学生目标会被拒绝。

### 学生不显示怎么办？

请检查该学生是否存在于权限清单中，并且 `role='student'`、`enabled=TRUE`。

### 保存标签失败怎么办？

请先确认：

1. 当前登录账号是管理员。
2. 目标账号是真实学生账号。
3. 后端服务和数据库连接正常。
4. 标签文本没有异常字符或超出业务预期。

## 管理建议

- 不要把管理员账号加入学生标签管理流程。
- 新增学生前先维护权限清单。
- 定期检查禁用学生账号是否还需要展示。
- 标签命名尽量简短稳定，便于 Reminder 和 Activity 复用。

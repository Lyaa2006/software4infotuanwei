# Web 学生账号范围与活动删除验证清单

本文件是开发者和测试同学使用的人工验证清单。它不包含密钥，不连接生产数据，也不会执行破坏性数据库操作。

## 验证目标

- 确认管理员账号不会被当作学生账号展示。
- 确认学生标签、党团进度、活动参与人和骨干设置都有后端保护。
- 确认前端页面对脏数据有防御性过滤。
- 确认班团活动删除功能入口清晰、权限合理、失败提示明确。

## 基础准备

1. 准备一个管理员账号。
2. 准备至少两个真实学生账号。
3. 准备一个班团骨干学生账号。
4. 准备一个非骨干普通学生账号。
5. 如有测试数据库，准备一个历史脏数据场景：`party_students` 中存在管理员账号行。
6. 不要在生产环境做破坏性删除验证。

## TAG 学生标签管理
### 涉及入口
- `/tag-management`
- `PUT /api/reminder/admin/students/:accountId/tags`

### 核心断言
- 页面列表只展示真实学生账号。
- 后端接口拒绝非学生目标。
- 管理员登录权限不受影响。
- 真实学生数据仍可正常维护。

### 用例 TAG-001: 管理员账号 / 正常数据
- 前置条件：使用 管理员账号，场景为正常数据。
- 操作入口：/tag-management, PUT /api/reminder/admin/students/:accountId/tags。
- 操作步骤：按页面入口打开功能，尝试查看列表、保存目标数据或触发相关操作。
- 期望结果：该账号不能作为学生管理对象；若直接调用学生对象接口，应返回 `NOT_STUDENT_ACCOUNT` 或明确拒绝信息。
- 复查点：刷新页面后结果保持；浏览器控制台不应出现未处理异常。
- 记录：填写测试账号、时间、结果、失败截图或接口响应。

```bash
curl -X PUT "$BASE/api/reminder/admin/students/$ADMIN_ACCOUNT/tags" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"tags":["班团骨干"]}'
```

期望：返回 `NOT_STUDENT_ACCOUNT`。

```bash

  -H "Authorization: Bearer $ADMIN_TOKEN"
```

期望：返回 `NOT_STUDENT_ACCOUNT`，且不会创建管理员的党团资料行。

### 活动参与人拒绝管理员


期望：返回 `INVALID_ACTIVITY_PARTICIPANT`。

## 回归风险清单

- 登录：管理员和学生登录都应不受影响。
- Reminder：发送给全部学生不应因为标签为空失败。
- Reminder：按标签发送仍需至少一个标签。
- TagManagement：真实学生标签保存后仍能刷新展示。
- PartyAdminEdit：党团日期顺序校验和未来日期校验不应被改坏。
- Activity：上传照片、编辑提案、提交审核仍可用。
- Activity：管理员审核通过/驳回仍可用。
- Activity：设为骨干/移除骨干仍可用。
- Activity：已通过活动不可编辑、不可删除。
- Cert/Honor：复用学生资料的页面遇到非学生 accountId 时应显示清晰错误，不应返回 500。
